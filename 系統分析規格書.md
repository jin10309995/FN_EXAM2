# 系統分析規格書

## 1. 文件目的
本文件描述升學與英語考題系統之業務需求、使用者角色、功能需求、非功能需求、資料需求與系統邊界，作為設計、開發、測試與驗收之依據。

## 2. 系統背景
本系統用於提供校內或補教場域之線上考題管理與測驗服務，支援從題庫建立、試卷組成、考生作答、成績分析到 AI 輔助出題的完整流程。

## 3. 名詞定義
| 名詞 | 定義 |
| --- | --- |
| 題庫 | 可重複使用之題目集合 |
| 試卷 | 由多道題目依序組成之測驗 |
| 作答紀錄 Submission | 學生對某一試卷之提交結果 |
| 作答明細 Answer Detail | 單一題目之答案與得分資訊 |
| 人工批改 | 對寫作、口說等題型由教師手動給分 |
| 題目封存 | 題目達指定條件後標記為 `is_archived=1`，不再優先使用 |

## 4. 系統目標
- 提供管理者維護題庫與試卷之能力
- 提供學生線上作答與成績查詢
- 提供各題正答率、難度、鑑別度與能力估計
- 提供寫作與口說題之人工批改
- 提供 AI 輔助出題與媒體素材管理

## 5. 利害關係人與角色
| 角色 | 需求摘要 |
| --- | --- |
| 系統管理者 | 管理題目、試卷、媒體、AI 出題與統計分析 |
| 教師/命題者 | 建立題目、檢查題目品質、人工批改 |
| 考生 | 瀏覽開放考試、作答、查詢成績與分析 |
| 維運人員 | 啟停系統、維護 `.env`、備份資料庫 |

## 6. 系統範圍
### 6.1 範圍內
- 題庫 CRUD
- 隨機選題與加權推薦
- 試卷 CRUD 與狀態控管
- 學生線上作答與自動評分
- 人工批改流程
- 成績查詢與分析
- AI 產題預覽與批次入庫
- 音訊與圖片上傳

### 6.2 範圍外
- 多校區多租戶管理
- 使用者帳號密碼登入
- 支付、課程管理、學籍整合
- 大型分散式部署與高可用叢集

## 7. 業務流程分析
### 7.1 題庫管理流程
1. 管理者進入後台。
2. 查詢或篩選題目。
3. 新增、修改、刪除或批次匯入題目。
4. 系統檢核必填欄位與題型/學段是否合法。
5. 寫入 `questions` 並回傳處理結果。

### 7.2 試卷管理流程
1. 管理者建立試卷基本資料。
2. 指派題目與配分。
3. 儲存為 `draft`、切換為 `active` 或 `closed`。
4. 學生僅可作答 `active` 試卷。

### 7.3 學生作答流程
1. 學生瀏覽考試列表。
2. 系統顯示開放中試卷。
3. 學生輸入姓名與學號後開始作答。
4. 系統依題型蒐集答案並倒數計時。
5. 提交後進行自動評分。
6. 系統建立 `submissions` 與 `answer_details`。
7. 顯示成績與分析結果。

### 7.4 人工批改流程
1. 管理者查詢待批改清單。
2. 檢視寫作或口說答案。
3. 輸入 `rubric_score` 與評語。
4. 系統更新作答明細狀態為 `graded`。
5. 系統重算 submission 總分。

### 7.5 AI 出題流程
1. 管理者輸入科目、題型、難度、題數、學段與提示。
2. 系統呼叫指定 LLM provider。
3. 回傳題目預覽資料。
4. 管理者審核後批次存入題庫。

## 8. 功能需求
### 8.1 題庫管理
- FR-01 系統應提供題目查詢、分頁、搜尋與篩選功能。
- FR-02 系統應支援題目新增。
- FR-03 系統應支援題目部分欄位更新。
- FR-04 系統應支援題目刪除。
- FR-05 系統應支援批次匯入題目。
- FR-06 系統應支援隨機選題與加權選題。

### 8.2 試卷管理
- FR-07 系統應提供試卷建立、查詢、修改與刪除功能。
- FR-08 系統應支援試卷關聯題目與配分。
- FR-09 系統應以 `draft/active/closed` 管理試卷狀態。
- FR-10 系統應將學生取題與管理者查卷分離，避免答案外洩。

### 8.3 作答與評分
- FR-11 系統應提供考生基本資料輸入。
- FR-12 系統應支援選擇、填空、計算、聽力、段落填空、閱讀、寫作、口說題型。
- FR-13 系統應提供倒數計時與交卷功能。
- FR-14 系統應於交卷後自動計分。
- FR-15 系統應記錄每題作答結果與得分。
- FR-16 系統應統計答對、答錯與不會次數。

### 8.4 成績與分析
- FR-17 系統應提供單次作答成績查詢。
- FR-18 系統應提供作答分析，含科目、難度、弱點題目與建議。
- FR-19 系統應提供試卷整體統計。
- FR-20 系統應提供題目難度校準、題目品質與學生能力分析。
- FR-21 系統應提供個人化推薦題目。

### 8.5 人工批改
- FR-22 系統應列出待批改作答明細。
- FR-23 系統應支援批改寫作與口說題。
- FR-24 系統應於批改後回寫總分。

### 8.6 AI 與媒體
- FR-25 系統應支援 AI 產題預覽。
- FR-26 系統應支援音訊上傳。
- FR-27 系統應支援圖片上傳。

## 9. 非功能需求
### 9.1 效能需求
- NFR-01 一般 API 每分鐘限制 200 次請求。
- NFR-02 提交作答每 15 分鐘限制 10 次。
- NFR-03 一般查詢應可於單機 SQLite 架構下正常回應。

### 9.2 安全需求
- NFR-04 管理功能需受 `x-api-key` 保護。
- NFR-05 學生取題 API 不得回傳正解。
- NFR-06 檔案上傳需限制檔案型態與大小。
- NFR-07 應使用安全 HTTP Header 與 CORS 控管。

### 9.3 可維護性需求
- NFR-08 資料表變更需透過 `database.js` migration 保持向後相容。
- NFR-09 設定需集中於 `.env`。
- NFR-10 前端頁面需可由 `generate-public.js` 重建。

### 9.4 可用性需求
- NFR-11 首頁應可導向主要使用流程。
- NFR-12 管理者應可透過單一後台操作題庫、試卷、分析與批改。

## 10. 角色使用案例
| 使用案例 | 角色 | 前置條件 | 成功結果 |
| --- | --- | --- | --- |
| UC-01 管理題目 | 管理者 | 具管理金鑰 | 題目新增/更新/刪除成功 |
| UC-02 建立試卷 | 管理者 | 題庫已有題目 | 試卷建立成功 |
| UC-03 學生作答 | 考生 | 試卷為 `active` | 建立作答紀錄與成績 |
| UC-04 查詢分析 | 考生/管理者 | 已有 submission | 顯示分析報告 |
| UC-05 人工批改 | 管理者 | 有 `pending` 題目 | 分數與評語寫回 |
| UC-06 AI 出題 | 管理者 | 已設定 API Key | 顯示預覽題目 |

## 11. 資料需求分析
### 11.1 主體資料
- 科目 `subjects`
- 題目 `questions`
- 試卷 `exams`
- 試卷題目關聯 `exam_questions`
- 作答紀錄 `submissions`
- 作答明細 `answer_details`

### 11.2 題目資料需求
- 基本欄位：科目、題型、難度、題幹、答案、詳解
- 選擇題欄位：選項 A-D
- 媒體欄位：音訊、音訊逐字稿、圖片
- GEPT 欄位：文章、段落編號
- 統計欄位：答對、答錯、不會、封存

### 11.3 作答資料需求
- 學生姓名
- 學號/座號
- 題目作答內容
- 是否正確
- 得分
- 批改狀態
- 評語

## 12. 業務規則
- BR-01 題型僅允許 `choice`、`fill`、`calculation`、`listening`、`cloze`、`reading`、`writing`、`speaking`
- BR-02 難度值範圍為 1 至 5
- BR-03 試卷狀態僅允許 `draft`、`active`、`closed`
- BR-04 僅 `active` 試卷可供學生作答
- BR-05 `writing` 與 `speaking` 需人工批改
- BR-06 `cloze` 題答案以 `|` 分隔逐格比對
- BR-07 學生可用 `__dont_know__` 表示不會，該題不計分並累計不會次數
- BR-08 題目答對次數超過 5 且未封存時，可自動封存並嘗試產生替換題

## 13. 介面需求
### 13.1 前端頁面
- `/` 首頁
- `/exam-list.html` 考試列表
- `/exam.html?id=...` 作答頁
- `/result.html?id=...` 單次成績
- `/analysis.html?id=...` 分析報告
- `/results.html` 成績查詢
- `/admin.html` 管理後台
- `/ai-generate.html` AI 出題頁

### 13.2 外部介接
- OpenAI API
- Google Gemini API
- Anthropic Claude API

## 14. 約束與假設
- 系統為單機單程序部署
- 主要資料庫為 SQLite
- 未內建完整會員系統，以 API Key 保護管理功能
- AI 生成題目品質仍需人工審查
- 現階段測試以手動驗證為主

## 15. 需求追蹤
| 需求分類 | 對應模組 |
| --- | --- |
| 題庫管理 | `server.js` 題目 API、`public/admin.html` |
| 試卷管理 | `server.js` 試卷 API、`public/admin.html` |
| 作答與評分 | `server.js` 提交 API、`public/exam.html` |
| 分析統計 | `server.js` 分析 API、`public/analysis.html` |
| AI 出題 | `llm.js`、`server.js` AI API、`public/ai-generate.html` |
| 媒體管理 | `multer` 上傳 API、`uploads/` |

## 16. 驗收重點
- 題目與試卷資料正確保存
- 學生作答後成績正確
- 人工批改後總分同步更新
- 分析報表可正確呈現
- AI 出題失敗時能回傳明確錯誤
- 未授權存取管理 API 時應回傳 401
