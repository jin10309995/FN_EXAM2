# 系統設計規格書

## 1. 文件目的
本文件說明升學與英語考題系統之技術架構、模組設計、資料庫設計、API 設計、安全設計與部署設計，作為開發與維護依據。

## 2. 系統架構
本系統採單體式三層近似架構：
- 表現層：`public/*.html` 靜態頁面與瀏覽器端 JavaScript
- 應用層：`server.js` 內的 Express 路由、評分與分析邏輯
- 資料層：`database.js` 中的 SQLite schema、migration 與 SQL 存取

## 3. 技術選型
| 類別 | 技術 |
| --- | --- |
| 語言/平台 | Node.js |
| Web Framework | Express 4 |
| 資料庫 | SQLite + `better-sqlite3` |
| 前端 | HTML + Tailwind CSS CDN |
| 上傳 | `multer` |
| 安全 | `helmet`、`cors`、`express-rate-limit` |
| AI | OpenAI、Gemini、Claude |

## 4. 模組設計
### 4.1 `server.js`
功能：
- 啟動 Express 服務
- 設定中介層與安全機制
- 提供 REST API
- 處理作答評分、人工批改、統計分析與推薦
- 提供媒體上傳與靜態檔案服務

### 4.2 `database.js`
功能：
- 建立資料庫連線
- 建立資料表
- 執行 migration
- 建立預設科目主檔

### 4.3 `llm.js`
功能：
- 封裝不同 LLM provider
- 建立系統提示詞
- 解析 JSON 回應

### 4.4 `generate-public.js`
功能：
- 重新產生 `public/` 靜態頁面
- 固化前端頁面樣板

### 4.5 `seed*.js`
功能：
- 匯入示範或正式題庫資料
- 按年級與考試類型進行分批建置

## 5. 執行流程設計
### 5.1 啟動流程
1. 載入 `.env`
2. 初始化資料庫與 migration
3. 建立上傳目錄
4. 設定中介層與靜態路徑
5. 註冊 API 路由
6. 啟動 HTTP 服務

### 5.2 交卷流程
1. 驗證基本輸入
2. 驗證試卷狀態為 `active`
3. 取回試卷題目與答案
4. 逐題評分並建立 answer detail
5. 在 transaction 中寫入 submission 與 answer_details
6. 更新題目答對/答錯/不會統計
7. 非同步執行題目封存與替換

### 5.3 人工批改流程
1. 查出待批改題目
2. 驗證評分輸入
3. 以 transaction 更新明細狀態與分數
4. 重算 submission 總分

## 8. 安全設計
- 使用 `helmet` 設定安全標頭
- 使用 `cors` 限制可接受來源
- 使用 `express-rate-limit` 控管濫用
- 管理 API 透過 `requireAdmin` 驗證 `x-api-key`
- 學生作答 API 與管理查卷 API 分離
- 錯誤處理中介層避免回傳 stack trace
- 檔案上傳限制：
  - 音訊 50MB
  - 圖片 10MB
  - 依 MIME 與副檔名驗證

## 9. 評分與分析設計
### 9.1 自動評分
- 選擇/填空/計算/聽力：字串比對
- `cloze`：`|` 拆分後逐格比對
- `writing`、`speaking`：建立 `pending` 明細，待人工批改
- `__dont_know__`：記錄為不會

### 9.2 題目統計
- 題目答對時累加 `correct_count`
- 答錯時累加 `wrong_count`
- 不會時累加 `dont_know_count`

### 9.3 題目品質分析
- 通過率
- 經驗難度
- 難度偏差
- 鑑別度指數
- 品質旗標

### 9.4 能力估計
- 採 Rasch 模型近似估算
- 題目標示難度 1-5 轉為 logit
- 回傳整體能力與分科能力

## 10. 題目封存與自動替換設計
- 觸發條件：`correct_count > 5` 且 `is_archived = 0`
- 處理流程：
  - 標記封存
  - 依原題型、科目、學段與較高難度呼叫 LLM
  - 產生替換題後寫入題庫
- 目的：降低題目重複曝光與題庫老化

## 11. 前端設計
### 11.1 頁面配置
- `index.html`：入口頁
- `exam-list.html`：顯示開放中考試
- `exam.html`：考試作答、音訊播放、圖片顯示、計時器
- `result.html`：成績頁
- `analysis.html`：分析頁
- `results.html`：成績查詢
- `admin.html`：管理整合頁
- `ai-generate.html`：AI 題目產生頁

### 11.2 前端特性
- Tailwind CSS CDN
- 純瀏覽器端呼叫 REST API
- 無前端框架與建置流程

## 12. 錯誤處理設計
- 參數缺失：HTTP 400
- 未授權：HTTP 401
- 找不到資料：HTTP 404
- 試卷狀態不允許提交：HTTP 403
- 金鑰未設定等服務不可用：HTTP 503
- 其他例外：HTTP 500

## 13. 部署設計
- 執行方式：
  - `npm install`
  - `node generate-public.js`
  - `node seed.js`
  - `node server.js`
- Windows 快速啟動：
  - `setup.bat`
  - `start.bat`
- 預設埠：`3000`

## 14. 設計限制
- 單體架構，所有商業邏輯集中於 `server.js`
- SQLite 適合中小規模使用
- 無獨立認證服務
- 無自動化測試框架

## 15. 可擴充設計建議
- 抽離 service/repository 層降低 `server.js` 複雜度
- 導入正式 migration 工具
- 導入登入與 RBAC
- 導入自動化測試與 CI
- 規劃 PostgreSQL 或 MySQL 升級路徑
