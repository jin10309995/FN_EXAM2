## 一、系統簡介

本系統為「升國中數理資優班甄試」線上考試平台，提供以下核心功能：

| 功能 | 說明 |
|------|------|
| 📚 題庫管理 | 建立、編輯、刪除各科目題目（選擇題、填充題、作文題） |
| 🤷 我不會 | 學生作答時可標記「我不會」，不計入分數，系統會優先於後續出題中加重此類題目 |
| 📝 試卷管理 | 組合試卷、設定時間與狀態（草稿／開放／關閉） |
| 🎓 線上作答 | 學生計時作答，系統自動評分，數學公式正確渲染 |
| 🔊 英語聽力 TTS | 聽力題以瀏覽器內建語音（Web Speech API）朗讀英文題目，無需音檔；最多播放 3 次，防止作答作弊 |
| ✍️ 作文作答 | 學生於作答頁填寫作文，顯示即時字數計算；提交後進入「人工批改」流程 |
| 📊 成績統計 | 查看個人成績、全班排行、錯誤率分析 |
| ✍️ 作文批改 | 教師後台批改學生作文，可一鍵呼叫 AI（Gemini/OpenAI/Claude）進行初步批改建議；AI 可產生示範範文供參考 |
| 🤖 AI 智慧出題 | 呼叫 LLM（OpenAI / Gemini 2.5 Flash / Claude）自動生成題目並批次儲存至題庫 |
| 📈 個人成績分析 | 依科目與難度分析作答表現，顯示弱點題目、個人化複習建議與 Rasch 能力估算 |
| 🔄 自動封存替換 | 答對次數超過門檻的題目自動封存，並由 AI 生成難度更高的替換題 |
| 🧠 ML 分析 | 題目品質分析（鑑別度指數）、難度校正（標示 vs. 實際）、學生能力剖面（IRT Rasch 模型）、個人化選題推薦 |

**支援科目：**
- **升國中（資優班）：** 數學、自然科學、物理、化學、生物、地球科學
- **國小六年級：** 數學、自然、國語、英語、英文聽力、社會、**作文**
- **國一（七年級）：** 國文、數學、英語、英文聽力、自然、社會、**作文**
- **國二（八年級）：** 國文、數學、英語、英文聽力、自然、社會、**作文**
- **國三（九年級）：** 國文、數學、英語、英文聽力、自然、社會、**作文**
- **國中教育會考：** 國文、數學、英語、英文聽力、自然、社會、**作文**（綜合三年題型）
- **全民英檢初級（GEPT）：** 聽力、閱讀、寫作、口說

**題目類型：** 選擇題（自動評分）、填充題（自動評分）、**作文／寫作（人工批改 + AI 初步批改）**  
**難度等級：** 1–5 星（入門 → 競賽級）  
**學段分類（選單順序）：** 國小六年級、升國中（資優班）、國一、國二、國三、國中教育會考、全民英檢初級

## 二、安裝與啟動

### 首次安裝（只需執行一次）

1. 確認已安裝 [Node.js](https://nodejs.org/) 16 以上版本
2. 在系統資料夾中，雙擊執行 `setup.bat`

```
setup.bat 會自動執行：
  [1/4] 安裝 Node.js 套件（npm install）
  [2/4] 產生前端頁面
  [3/4] 初始化資料庫
  [4/4] 植入範例題目
```

3. 出現「設定完成！請執行 start.bat 啟動系統」即表示安裝成功

### 日常啟動

雙擊 `start.bat`，出現以下訊息即表示啟動成功：

```
🎓 升國中數理資優班考題系統
   http://localhost:3000
```

開啟瀏覽器，前往 **http://localhost:3000** 即可使用。

> ⚠️ **注意：** 伺服器視窗不可關閉，關閉後系統停止運作。

### 關閉系統

直接關閉 `start.bat` 所開啟的命令提示字元視窗即可。

## 四、學生端操作

### 4.1 選擇考試

1. 點選首頁「**📝 參加考試**」
2. 進入考試列表頁，可看到目前**開放中**的試卷
3. 每張試卷卡片顯示：
   - 試卷名稱與說明
   - 考試時間（分鐘）
   - 題目數量與總分
4. 點選欲參加的試卷，進入考生資料填寫頁

> ℹ️ 只有狀態為「**開放中（active）**」的試卷才會顯示。草稿或已關閉的試卷不會出現。

---

### 4.2 填寫考生資料

進入試卷後，需填寫以下資料：

| 欄位 | 必填 | 說明 |
|------|------|------|
| **姓名** | ✅ 必填 | 請輸入真實姓名，成績單將顯示此名稱 |
| **座號／學號** | 選填 | 可填入班級座號或學號以便查詢 |

填寫完畢後，點選「**開始考試**」按鈕進入作答畫面。

---

### 4.3 進行作答

進入作答畫面後，介面包含以下區域：

```
┌──────────────────────────────────────────┐
│  試卷標題        剩餘時間 [MM:SS] [提交]  │
├──────────────────────────────────────────┤
│                                          │
│  第 1 題（選擇題）                        │
│  題目內容文字...                          │
│                                          │
│  ○ A. 選項一                             │
│  ○ B. 選項二                             │
│  ○ C. 選項三                             │
│  ○ D. 選項四                             │
│                                          │
│  第 2 題（填充題）                        │
│  題目內容... ＿＿＿                       │
│  [請填寫答案                          ]  │
│                                          │
├──────────────────────────────────────────┤
│  題目快速導覽：[1][2][3][4]...[N]        │
└──────────────────────────────────────────┘
```

#### 作答說明

| 題目類型 | 操作方式 |
|----------|----------|
| **選擇題** | 點選 A / B / C / D 選項的圓形按鈕 |
| **填充題** | 在輸入框中直接輸入答案文字 |
| **英文聽力題** | 點選「🔊 朗讀題目」按鈕收聽英文朗讀，最多播放 **3 次**；依題型選擇或填寫答案 |

> 💡 **聽力題注意事項：**  
> - 請確認裝置**音量已開啟**，並使用耳機或喇叭在安靜環境作答  
> - 朗讀功能由瀏覽器內建語音引擎（Web Speech API）提供，建議使用 **Chrome 或 Edge** 瀏覽器  
> - 播放 3 次後按鈕停用，無法再次收聽，請仔細聆聽
> - 點選「📄 顯示文字」可展開朗讀內容的文字版本，方便不熟悉英文聽力的學生輔助閱讀

#### 「🤷 我不會」功能

每道題目右下角有「**🤷 我不會**」按鈕，供學生在不確定答案時使用：

| 狀況 | 效果 |
|------|------|
| 點選「🤷 我不會」 | 該題標記為「不會」，分數計為 0，**不算答錯** |
| 若先選「我不會」後又作答 | 可直接作答，「我不會」標記自動取消 |
| 成績頁 | 顯示「🤷 我不會」標記，獨立於答對／答錯之外 |

> 💡 建議學生在真的不知道答案時使用此功能，而非亂猜。選「我不會」後，後續**隨機出題**時系統會以**更高權重**優先抽選這類題目，幫助針對弱點加強練習。

#### 計時器

- 右上角顯示**倒數計時**（格式：MM:SS）
- 時間到 **0:00** 時系統自動提交，無需手動操作
- 建議在時間結束前確認作答完整性

#### 題目導覽列

- 畫面下方顯示所有題號按鈕
- 點選題號可快速跳至該題
- 已作答的題目按鈕會改變顯示樣式

---

### 4.4 提交與查看成績

#### 手動提交

點選右上角「**提交作答**」→ 確認提示視窗 → 點選「**確定**」

> ⚠️ 提交後無法修改，請確認作答完畢再提交。

#### 成績頁面

提交後自動跳轉至個人成績頁，顯示：

```
┌──────────────────────────────┐
│      王小明 的成績            │
│                              │
│           85%                │
│   85 / 100 分 · 作答時間     │
│                              │
│  ✅ 優秀！繼續保持！          │
└──────────────────────────────┘
```

- **百分比顯示**：依得分比例以顏色標示（綠色：優秀 / 黃色：良好 / 橘色：加油）
- **每題詳解**：顯示每一題的題目、你的答案、正確答案及詳細解析
- 標記 ✓ **答對**（+得分）或 ✗ **答錯**

#### 成績頁面網址格式

```
http://localhost:3000/result.html?id=<作答ID>
```

可將此網址分享給家長查看成績。

---

### 4.5 個人成績分析

提交作答後，可在成績頁面點選「**📈 詳細分析**」按鈕，進入個人成績分析頁面：

```
http://localhost:3000/analysis.html?id=<作答ID>
```

#### 分析內容說明

| 分析項目 | 說明 |
|----------|------|
| **總覽** | 得分、百分比、答對/答錯題數 |
| **科目分布** | 各科目的答對／答錯題數與得分統計（橫條圖） |
| **難度分布** | 各難度等級（1–5 星）的答題狀況表格 |
| **🧠 能力估算** | 以 Rasch IRT 模型估算各科能力值（1–5），並顯示綜合能力值 |
| **弱點題目** | 列出所有答錯的題目，含正確答案與詳解 |
| **個人化建議** | 依弱點科目與平均難度，自動產生複習建議 |

#### 建議說明範例

- 「建議加強：數學 的練習。」（弱點科目）
- 「答錯題目平均難度為 2.5，建議從此難度附近的題目開始複習。」
- 「整體表現良好，繼續加油！」（依百分比分段給予鼓勵）

#### 能力估算說明（Rasch IRT）

作答結束後系統自動以 **Rasch 模型**（Item Response Theory）估算學生的各科能力值：

- 能力值範圍：**1.0（最低）～ 5.0（最高）**，與題目難度（1–5 星）同一尺度
- 能力值顯示為橫條圖，顏色代表程度：🟢 綠色（≥4.0）、🟡 黃色（≥2.5）、🔴 紅色（<2.5）
- 樣本數越多（即作答題目越多）估算越準確

> 💡 建議學生在每次考試後查看個人分析，針對弱點科目進行加強複習。

## 五、教師端操作（後台管理）

### 5.1 進入後台

點選首頁「**⚙️ 後台管理**」，或直接前往：

```
http://localhost:3000/admin.html
```

後台分為四個功能頁籤：

| 頁籤 | 功能 |
|------|------|
| **題庫管理** | 新增、編輯、刪除題目 |
| **試卷管理** | 建立試卷、組合題目、控制開放狀態 |
| **成績統計** | 查看各試卷統計與錯誤率分析 |
| **🧠 ML 分析** | 題目品質分析、難度校正、學生能力查詢 |

---

### 5.2 題庫管理

點選上方「**題庫管理**」頁籤（預設顯示）。

#### 篩選與搜尋

頁面上方提供五種篩選條件，可組合使用：

| 篩選項目 | 說明 |
|----------|------|
| **全部科目** 下拉選單 | 篩選特定科目（數學、物理…） |
| **全部類型** 下拉選單 | 篩選選擇題 / 填充題 |
| **全部難度** 下拉選單 | 篩選 1–5 星難度 |
| **全部學段** 下拉選單 | 篩選「國小六年級」或「升國中（資優班）」 |
| **搜尋框** | 輸入關鍵字即時搜尋題目內容或標籤 |

#### 新增題目

1. 點選右上角「**＋ 新增題目**」按鈕
2. 填寫題目表單：

| 欄位 | 必填 | 說明 |
|------|------|------|
| **科目** | ✅ | 選擇題目所屬科目；選擇「作文」科目時，題型自動鎖定為「寫作」 |
| **題型** | ✅ | 選擇題 / 填充題 / 寫作（作文科目時自動設定） |
| **難度** | ✅ | 1（入門）至 5（競賽級） |
| **學段** | ✅ | 國小六年級 / 升國中（資優班）/ 國一 / 國二 / 國三 / 國中教育會考 / 全民英檢初級 |
| **題目內容** | ✅ | 輸入題目文字，可包含數學符號 |
| **選項 A/B/C/D** | 選擇題必填 | 填入各選項文字 |
| **TTS 朗讀文字** | 英文聽力科目必填 | 學生作答時由瀏覽器語音朗讀的英文文字，可點選「🔊 預覽 TTS 朗讀」試聽 |
| **正確答案 / 評分規準** | 選填 | 選擇題填 A/B/C/D；填充題填正確答案文字；**作文題填評分規準**（如「立意 40%、結構 30%、語文 30%」），AI 批改時會參照此欄位 |
| **解析** | 選填 | 詳細解題說明，作答後顯示給學生 |
| **來源** | 選填 | 題目出處（如：XX年份考題） |
| **標籤** | 選填 | 以逗號分隔，方便搜尋（如：數論,質數） |

3. 若已填好題目內容，可點選「**AI 產生答案與解析**」：

| 項目 | 說明 |
|------|------|
| **用途** | 依目前表單中的題目內容，自動回填「正確答案」與「解析」欄位 |
| **前置條件** | 伺服器 `.env` 已設定至少一組 LLM API Key，且伺服器已重新啟動 |
| **適用題型** | 選擇題、填空題、閱讀理解、段落填空、寫作、口說 |
| **選擇題限制** | 必須先完整填入 A/B/C/D 四個選項 |
| **聽力題限制** | 建議先填寫「TTS 朗讀文字」，讓 AI 可根據完整題意判斷 |
| **結果處理** | 系統會自動將產生結果填入「正確答案」與「解析」欄位，教師仍可手動修改 |

4. 確認答案與解析內容正確後，點選「**儲存**」完成新增

> ⚠️ AI 產生的答案與解析屬於**參考結果**，仍應由教師人工覆核後再發佈。

#### 編輯題目

在題目列表中，點選該題右側的「**編輯**」按鈕，修改後點選「**儲存**」。

#### 刪除題目

在題目列表中，點選該題右側的「**刪除**」按鈕，確認後刪除。

> ⚠️ 刪除後無法復原，請確認該題未被試卷使用。

#### 題目列表欄位說明

| 欄位 | 說明 |
|------|------|
| **ID** | 系統自動編號 |
| **科目** | 題目所屬科目 |

#### 隨機抽題

此功能可依條件自動從題庫中隨機選取指定數量的題目，適合快速組卷。試卷編輯畫面中有獨立的「**🎲 隨機抽題**」區塊，位於「選擇題目」清單下方。

1. （選用）在 🎲 隨機抽題區塊設定篩選條件：

| 步驟 | 操作 | 效果 |

### 5.6 ML 分析（機器學習）

點選上方「**🧠 ML 分析**」頁籤，進入機器學習分析模組。包含三個子功能：

> ⚠️ ML 分析頁籤需要管理員金鑰（若系統有設定 `ADMIN_API_KEY`）才能存取 API。

---

#### 🔍 題目品質分析

**功能說明**：分析題庫中已有足夠作答紀錄的題目，計算統計品質指標，協助教師找出需要改善的題目。

**操作步驟：**

1. 設定「**最少作答次數**」門檻（預設 3 次，次數越高資料越可靠）
2. 選擇「**學段**」（全部 / 升國中 / 國小六年級）
3. 點選「**載入報表**」
4. 可勾選「**只顯示需審查**」快速篩選異常題目

**指標說明：**

| 指標 | 說明 | 判斷標準 |
|------|------|----------|
| **通過率（Pass Rate）** | 答對人數 / 總作答人數 | 正常範圍：20%–95% |
| **鑑別度（Discrimination Index）** | 高分組通過率 − 低分組通過率 | ≥0.4 佳；<0.2 低；<0 為負鑑別（異常） |
| **品質分（Quality Score）** | 綜合評分 0–100 | 70 以上正常；越低越需審查 |
| **問題旗標** | 自動偵測到的品質問題 | 見下表 |

**可能出現的旗標：**

| 旗標 | 說明 | 建議處理 |
|------|------|----------|
| 太容易（通過率>95%） | 幾乎所有人都答對，缺乏區分效果 | 提高難度或刪除 |
| 太困難（通過率<5%） | 幾乎所有人都答錯，可能超出範圍 | 降低難度或確認題意 |
| 鑑別度低（<0.2） | 高低分者作答結果差異不大 | 檢視題目設計是否清晰 |
| 負鑑別度 | 低分者反而比高分者更容易答對 | 立即審查，可能有答案錯誤 |
| 難度標示異常 | 實際通過率與標示難度偏差 ≥2 級 | 更新題目難度標籤 |

---

#### 📊 難度校正

**功能說明**：比較題目的「標示難度（1–5 星）」與「實際通過率推算的難度」，找出難度標示不準確的題目。

**操作步驟：**

1. 選擇「**學段**」
2. 點選「**載入報表**」
3. 可勾選「**只顯示異常題目**」（偏差 ≥2 級且有足夠資料）

**欄位說明：**

| 欄位 | 說明 |
|------|------|
| **標示難度** | AI 或教師當初設定的 1–5 星難度 |
| **實際難度** | 依實際通過率推算的難度（≥80% → 1 星；<20% → 5 星） |
| **偏差** | 實際 − 標示，正值代表「比標示更難」，負值代表「比標示更容易」 |
| **作答數** | 題目被作答的總次數（需 ≥5 次資料才可靠） |

> 💡 建議每學期執行一次難度校正，確保題庫難度標示的準確性。

---

#### 🧠 學生能力查詢

**功能說明**：輸入學生姓名，系統以 **Rasch IRT（Item Response Theory）**模型，根據該學生所有歷次考試的作答紀錄，估算其各科能力值。

**操作步驟：**

1. 在「**學生姓名**」欄位輸入學生的姓名（需與考試時填寫的完全一致）
2. 可選填「**學生編號**」進一步確認身份
3. 點選「**查詢**」

**結果說明：**

```
王小明
共 3 份考卷 · 45 題作答紀錄

各科能力分布：
數學    ████████░░  3.8 / 5  （20 題）
物理    ██████░░░░  3.1 / 5  （15 題）
化學    ████░░░░░░  2.4 / 5  （10 題）

綜合能力值：3.2 / 5
```

| 能力值 | 顏色 | 說明 |
|--------|------|------|
| 4.0 以上 | 🟢 綠色 | 優秀，可挑戰更高難度 |
| 2.5–3.9 | 🟡 黃色 | 中等，仍有進步空間 |
| 2.4 以下 | 🔴 紅色 | 基礎薄弱，需加強練習 |

> ℹ️ Rasch 模型以「答對難度 N 的題目比答錯難度 N 的題目更多」等資訊，推算學生能力值。單次考試樣本數較少時（< 10 題），估算值僅供參考。

#### API 個人化推薦端點

後端另提供以下 API 供進階整合使用：

```
GET /api/recommendations?student_name=王小明&count=10
```

| 參數 | 說明 |
|------|------|
| `student_name` | 學生姓名（依歷史紀錄推薦）|
| `student_id` | 學生編號（可與姓名並用）|
| `subject_id` | 指定科目 ID（選填）|
| `grade_level` | 學段篩選（`junior_high` / `elementary_6`，選填）|
| `count` | 推薦題數（預設 10，最大 50）|

系統會依學生能力估算值，自動推薦「能力值 + 0.5」難度附近的弱點科目題目，並排除近期已作答過的題目。若學生無作答紀錄則隨機推薦。

## 六、AI 智慧出題

AI 出題功能可呼叫大型語言模型（LLM）自動生成考題，大幅節省手動出題時間。生成的題目經教師審核後，可批次儲存至題庫直接使用。

### 6.1 設定 LLM API Key

AI 出題功能需要在伺服器端設定 LLM API Key，**前端頁面上的「管理員金鑰」欄位與 LLM 金鑰無關**。

**設定步驟：**

1. 在系統資料夾中找到 `.env` 檔案（若不存在請複製 `.env.example`）
2. 依選用的服務填入對應金鑰：

| LLM 服務 | 環境變數 | 金鑰格式範例 |
|----------|----------|--------------|
| **OpenAI**（GPT-4o-mini） | `OPENAI_API_KEY` | `sk-proj-...` |
| **Google Gemini 2.5 Flash** | `GEMINI_API_KEY` | `AIza...` |
| **Anthropic Claude** 3.5 Haiku | `ANTHROPIC_API_KEY` | `sk-ant-...` |

3. 儲存 `.env` 後，**重新啟動伺服器**（`start.bat`）即可使用

> ⚠️ 若 API Key 未設定，呼叫時系統回傳「503 服務不可用」並說明原因。

**設定面板欄位說明：**

| 欄位 | 說明 |

**操作步驟：**

1. 填寫出題設定
2. 點選「**🤖 產生題目**」按鈕
3. 等待 LLM 回應（通常 3–15 秒）
4. 預覽生成的題目

---

### 6.3 預覽與批次儲存

生成完成後，頁面下方顯示題目預覽卡片，每張卡片包含：
- 題目內容
- 選項（選擇題）
- 正確答案
- 解析

**審核與儲存流程：**

1. 閱讀每道題目，確認內容正確無誤
2. 點選不滿意的題目右上角 **✕** 移除
3. 確認後點選「**💾 批次儲存至題庫**」
4. 系統回傳儲存成功的題目數量與 ID

> ⚠️ **注意事項：**
> - AI 生成的題目可能存在錯誤，**務必逐題審核後再儲存**
> - 批次儲存後可在「後台管理 → 題庫管理」中繼續編輯或修正

### 6.4 單題答案與解析生成

除了「AI 出題」頁面可整批生成題目外，後台「**題庫管理 → 新增題目**」視窗也提供單題輔助功能。

**操作流程：**

1. 進入後台管理，點選「**＋ 新增題目**」
2. 先填寫學段、科目、題型、難度與題目內容
3. 若為選擇題，請先填好 A/B/C/D 四個選項
4. 點選「**AI 產生答案與解析**」
5. 系統會自動回填：
   - **正確答案**
   - **解析**
6. 教師確認內容後，再點選「**儲存**」

**常見失敗原因：**

| 狀況 | 原因與處理方式 |
|------|----------------|
| 顯示「請先填寫題目內容」 | 題目內容尚未輸入 |
| 顯示「選擇題請先完整填寫 A/B/C/D 選項」 | 選項未填完整 |
| 顯示 `503` 或 API Key 未設定 | `.env` 尚未設定 LLM 金鑰，或修改後未重啟伺服器 |
| 顯示 `Connection error` / `fetch failed` | 伺服器無法連到外部 LLM 服務，請檢查網路、防火牆或服務商狀態 |

> ⚠️ 此功能與「AI 出題」共用同一組 LLM 設定。若 AI 出題不能使用，單題答案/解析生成通常也會一併失敗。

### 自動封存與替換機制

系統會在每次學生提交作答後，自動檢查題庫中**答對次數超過 5 次**的題目：

1. 自動將該題目標記為「**已封存（is_archived）**」，不再出現於隨機抽題
2. 呼叫 LLM 自動生成一道**難度 +1** 的替換題（需設定 LLM API Key）
3. 替換題來源欄位標記為「自動替換（原題 #ID）」

> ℹ️ 此機制確保高曝光的題目持續汰換，維持題庫新鮮度。
> ⚠️ 若 LLM API Key 未設定，仍會執行封存，但不會自動生成替換題（伺服器 console 會顯示警告）。

**已封存題目的篩選：**
- 後台題庫管理頁面預設**隱藏**已封存題目
- API 查詢可加上 `include_archived=1` 參數顯示所有題目（含封存）



資料庫檔案位於系統資料夾中的 `exam.db`。

**備份方式：**

1. 關閉伺服器（關閉 start.bat 視窗）
2. 複製 `exam.db` 到安全位置
3. 重新啟動伺服器

> ⚠️ 請勿在伺服器運作中直接複製資料庫，可能導致資料不完整。

**使用 DB Browser 查看資料庫：**

下載 [DB Browser for SQLite](https://sqlitebrowser.org/)，開啟 `exam.db` 可直接瀏覽和匯出資料。

## 九、常見問題

**Q1：學生看不到任何試卷？**  
→ 確認試卷狀態已設為「**開放中（active）**」。草稿與已關閉的試卷不會顯示。

**Q2：伺服器無法啟動，出現「address already in use」？**  
→ 埠 3000 已被其他程式佔用。請關閉其他使用 3000 埠的程式，或以 `set PORT=3001 && node server.js` 使用其他埠。

**Q3：填充題答案輸入後沒有得分？**  
→ 系統以**完全比對**方式評分（不區分大小寫）。請確認正確答案設定時的格式與學生可能輸入的格式一致。例如：答案設定為 `H₂O`，學生若輸入 `h2o` 則不符。建議在解析欄說明標準答案格式。

**Q4：如何刪除學生的作答紀錄？**  
→ 目前系統不提供前端介面刪除單筆作答紀錄。如有需要，可使用 [DB Browser for SQLite](https://sqlitebrowser.org/) 開啟 `exam.db`，在 `submissions` 資料表中手動刪除。

**Q5：考試時間到了學生還沒提交會怎樣？**  
→ 倒數歸零後系統會**自動提交**目前的作答內容，學生無需手動操作。

**Q6：可以讓同一位學生重複作答嗎？**  
→ 可以。系統允許同一姓名多次提交，每次均視為獨立作答紀錄，成績列表中會顯示所有紀錄。

**Q7：題庫有多少題？**  
→ 系統目前內建超過 **4,900 道題目**，涵蓋升國中資優班、國小六年級、國一至國三、國中教育會考及全民英檢初級各學段與科目（含各年段英文聽力題庫各 100 題）。教師可透過後台持續新增題目。

**Q8：如何匯出成績報表？**  
→ 目前系統不內建匯出功能。可開啟瀏覽器的「**另存為 HTML**」儲存成績頁面，或使用 DB Browser 將 `submissions` 資料表匯出為 CSV 檔案。

**Q9：如何新增國小六年級題目？**  
→ 進入後台管理 → 題庫管理 → 點選「**＋ 新增題目**」→ 在「**學段**」欄位選擇「**國小六年級**」，科目可選擇數學、自然、國語、英語、英文聽力或社會。

**Q10：如何只看某一學段的題目？**  
→ 在後台管理「題庫管理」頁面，點選「**全部學段**」下拉選單，選擇「國小六年級」或「升國中」即可篩選顯示。

**Q11：組卷時可以混合兩個學段的題目嗎？**  
→ 可以。試卷選題時不限制學段，可同時勾選國小六年級與升國中的題目組成同一份試卷。若要分開出卷，在選題區的「全部學段」下拉選單選擇特定學段即可快速篩選。

**Q12：AI 出題功能如何使用？需要付費嗎？**  
→ 需先在 `.env` 設定所選 LLM 服務的 API Key，重新啟動伺服器後即可使用。API 費用依各服務商收費（OpenAI / Google Gemini / Anthropic），通常每次出題費用極低。支援的服務中 **Google Gemini** 提供免費額度，可優先試用。頁面預設選擇 **Gemini 2.5 Flash**。設定後前往 `http://localhost:3000/ai-generate.html` 使用。

**Q13：題目被「封存」後還能看到嗎？**  
→ 封存的題目預設不出現在隨機抽題中，後台題庫管理也會隱藏。但題目資料並未刪除，可透過 API 查詢（加 `include_archived=1` 參數）或使用 DB Browser 開啟 `exam.db` 直接查看 `questions` 資料表（`is_archived=1`）。如需還原，可在資料庫中將 `is_archived` 設回 `0`。

**Q14：ML 分析功能需要有足夠的作答資料嗎？**  
→ 是的。題目品質分析預設需要至少 3 次作答才納入分析（可調整），鑑別度指數需要至少 6 次才計算。學生能力查詢則需要該學生有至少一次作答紀錄。資料越多、估算越準確。

**Q15：Rasch 能力估算值代表什麼意思？**  
→ 能力值（1–5）與題目難度（1–5 星）使用相同尺度。能力值 3.0 代表該學生在難度 3 的題目上預期有約 50% 正確率；能力值 4.0 表示在難度 4 題目上有 50% 正確率，以此類推。能力值越高代表學生越優秀。

**Q16：題目數學公式顯示不正確怎麼辦？**  
→ 系統使用 MathJax 渲染 LaTeX 數學公式。題目請使用 `$...$`（行內公式）或 `$$...$$`（獨立公式）包住 LaTeX 語法，例如：`$\frac{1}{2}$`、`$x^2 + y^2 = r^2$`。若頁面未渲染，請確認網路可連線（MathJax 從 CDN 載入）。

**Q17：英語聽力題的「TTS 朗讀」功能無法使用？**  
→ TTS 朗讀使用瀏覽器內建的 Web Speech API，建議使用 **Chrome 或 Edge**。若朗讀無聲，請確認：(1) 系統音量已開啟；(2) 瀏覽器頁面允許播放音訊（地址欄附近的音訊圖示）；(3) 重新整理頁面後再試。iOS Safari 也支援，但語音音色可能不同。

**Q18：如何在後台新增一道英文聽力題？**  
→ 進入後台 → 點選「**＋ 新增題目**」→ 選擇學段（如：國一）→ **科目選「英文聽力」** → 題型選「選擇題」或「填充題」→ 輸入題目問題描述 → 在「**TTS 朗讀文字**」欄填入要朗讀的英文內容（如：`The cat is under the table.`）→ 可點「**🔊 預覽 TTS 朗讀**」試聽效果 → 填入選項與正確答案後儲存。

**Q19：「我不會」按鈕有什麼用途？**  
→ 學生作答時，若真的不知道答案，可點選題目右下角的「**🤷 我不會**」按鈕代替作答。效果如下：  
- 該題**不計入得分**（得 0 分），但**不標記為答錯**  
- 成績頁面顯示「🤷 我不會」標記，獨立於答對／答錯統計之外  
- 後續**隨機出題時，系統以雙倍權重優先抽選**這類題目，幫助學生針對弱點加強複習  
建議學生在真的不懂時才使用此功能，而非亂猜，以確保學習成效的真實反映。
