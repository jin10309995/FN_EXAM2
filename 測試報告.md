## 四、發現問題與修復紀錄

### Bug #1 — 隨機抽題路由衝突（已修復）

| 項目 | 說明 |
|------|------|
| **嚴重度** | 高 |
| **症狀** | `GET /api/questions/random` 回傳 404 |
| **根本原因** | `GET /api/questions/:id` 路由定義在 `/api/questions/random` 之前，Express 將 `"random"` 誤判為 id 參數，找不到對應題目而回傳 404 |
| **修復方式** | 將 `/api/questions/random` 路由移至 `/api/questions/:id` **之前**宣告 |
| **修復檔案** | `server.js` |

### Bug #2 — 刪除試卷外鍵約束失敗（已修復）

| 項目 | 說明 |
|------|------|
| **嚴重度** | 高 |
| **症狀** | `DELETE /api/exams/:id` 回傳 500，錯誤：`FOREIGN KEY constraint failed` |
| **根本原因** | `submissions` 資料表的 `exam_id` 外鍵未設定 `ON DELETE CASCADE`，刪除試卷時因有關聯作答紀錄而失敗 |
| **修復方式** | 在 `DELETE /api/exams/:id` 路由中，先刪除關聯的 `answer_details` 及 `submissions`，再刪除 `exams` |
| **修復檔案** | `server.js` |

### Bug #3 — PUT 路由不支援部分欄位更新（已修復）

| 項目 | 說明 |
|------|------|
| **嚴重度** | 高 |
| **症狀** | `PUT /api/questions/:id` 及 `PUT /api/exams/:id` 只傳入部分欄位時回傳 500 |
| **根本原因** | UPDATE SQL 直接綁定傳入值，未傳入的欄位為 `undefined` → SQLite 收到 `NULL` → 觸發 NOT NULL 約束失敗 |
| **修復方式** | 將 UPDATE 中的必填欄位改為 `COALESCE(?, 欄位名)` 模式，使未傳入的欄位保留原值；並為 `PUT /api/exams/:id` 補上 404 檢查 |
| **修復檔案** | `server.js` |
