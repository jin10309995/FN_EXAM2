### H-1｜SQL Injection — 字串拼接（高風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A03:2021 – Injection |
| **CVSS 評分** | 7.5（高） |
| **受影響端點** | `DELETE /api/exams/:id` |
| **修復狀態** | ✅ 已修復 |

**問題描述**

`DELETE /api/exams/:id` 在刪除試卷前，需先刪除關聯的作答明細。原始程式碼將資料庫查詢出的 submission ID 陣列直接以字串拼接方式組成 SQL `IN` 子句：

```javascript
// ❌ 修復前：危險的字串拼接
const subIds = db.prepare('SELECT id FROM submissions WHERE exam_id = ?')
                 .all(req.params.id).map(s => s.id);
if (subIds.length) {
  db.prepare(`DELETE FROM answer_details WHERE submission_id IN (${subIds.join(',')})`)
    .run(); // subIds 直接拼入 SQL！
}
```

雖然 `subIds` 來自資料庫查詢，但此寫法為危險模式，若資料庫中 submission ID 欄位遭污染（二次注入），可能導致任意 SQL 執行。此外，此模式若被複製到其他地方而 ID 來源改為使用者輸入，將直接造成 SQL Injection。

**攻擊情境**

```
攻擊者 → DELETE /api/exams/1
若 submission.id 含惡意內容 → SQL 被竄改 → 任意資料庫操作
```

**修復方式**

改用 SQL 子查詢，完全消除字串拼接，並包裝於 Transaction 中：

```javascript
// ✅ 修復後：子查詢 + Transaction
const deleteExam = db.transaction((id) => {
  db.prepare(`DELETE FROM answer_details
              WHERE submission_id IN (SELECT id FROM submissions WHERE exam_id = ?)`)
    .run(id);
  db.prepare('DELETE FROM submissions WHERE exam_id = ?').run(id);
  db.prepare('DELETE FROM exams WHERE id = ?').run(id);
});
deleteExam(req.params.id);
```

### H-2｜無身份驗證機制（高風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A01:2021 – Broken Access Control |
| **CVSS 評分** | 9.1（嚴重） |
| **受影響端點** | 所有管理員 API（POST/PUT/DELETE） |
| **修復狀態** | ✅ 已修復 |

**問題描述**

系統所有 API 端點完全對外開放，無任何驗證或授權機制。任何人只要能連上伺服器，皆可：
- 新增、修改、刪除全部題目與試卷
- 查看所有學生作答紀錄與個人資料
- 關閉或竄改進行中的考試

**受影響路由（修復前均無保護）**

```
POST   /api/questions              新增題目
PUT    /api/questions/:id          修改題目
DELETE /api/questions/:id          刪除題目
POST   /api/exams                  建立試卷
PUT    /api/exams/:id              修改試卷（含開放/關閉）
DELETE /api/exams/:id              刪除試卷
GET    /api/exams/:id/submissions  查看所有學生成績
GET    /api/exams/:id/stats        查看統計資料
GET    /api/submissions/:id        查看個人作答明細
```

**修復方式**

加入管理員 API 金鑰中介層，透過環境變數設定金鑰：

```javascript
// ✅ 修復後：管理員金鑰中介層
function requireAdmin(req, res, next) {
  const adminKey = process.env.ADMIN_API_KEY;
  if (adminKey) {
    const key = req.headers['x-api-key'];
    if (!key || key !== adminKey)
      return res.status(401).json({ error: '未授權，需要管理員金鑰' });
  }
  next();
}

// 所有管理 API 套用：
app.post('/api/questions', requireAdmin, ...);
app.delete('/api/exams/:id', requireAdmin, ...);
// ...
```

金鑰設定於 `.env` 檔案：
```
ADMIN_API_KEY=your-random-secret-key
```

> 未設定 `ADMIN_API_KEY` 時維持開放模式，適合本機單機使用。部署對外環境時**務必設定**。

### H-3｜考試答案洩漏（高風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A01:2021 – Broken Access Control |
| **CVSS 評分** | 8.2（高） |
| **受影響端點** | `GET /api/exams/:id` |
| **修復狀態** | ✅ 已修復 |

**問題描述**

系統提供兩個取得試卷的端點：
- `/api/exams/:id/take`：學生作答用，**有隱藏答案**
- `/api/exams/:id`：管理員用，**包含完整答案**

修復前，`/api/exams/:id` 完全公開，學生只需在瀏覽器開啟此 URL，即可取得所有題目的正確答案，完全繞過考試機制。

**攻擊情境**

```
1. 學生得知試卷 ID = 1
2. 直接呼叫：GET /api/exams/1
3. 回應包含每題的 answer 欄位
4. 學生取得全部答案後再進行作答 → 100% 得分
```

**驗證（修復後）**

```javascript
// 修復後測試：/take 端點不含答案欄位
const take = await fetch('/api/exams/1/take');
const data = await take.json();
data.questions.forEach(q => {
  console.log('answer' in q); // false ✅
});
```

**修復方式**

為 `GET /api/exams/:id` 加上 `requireAdmin` 中介層保護：

```javascript
// ✅ 修復後
app.get('/api/exams/:id', requireAdmin, (req, res) => { ... });
```

### H-4｜無速率限制（可暴力猜測答案）（高風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A04:2021 – Insecure Design |
| **CVSS 評分** | 7.3（高） |
| **受影響端點** | `POST /api/exams/:id/submit` |
| **修復狀態** | ✅ 已修復 |

**問題描述**

提交作答 API 無任何速率限制，且系統允許相同學生無限次提交同一份試卷。攻擊者可透過腳本：
1. **暴力窮舉答案**：一份 10 題選擇題試卷，最多 4¹⁰ = 1,048,576 次即可找出全部正確答案
2. **成績洗分**：提交多次，保留最高分紀錄
3. **DoS 攻擊**：大量請求造成伺服器過載

**攻擊概念**

```python
# 概念示意（偽代碼）
for answers in all_combinations(['A','B','C','D'], n=10):
    response = post('/api/exams/1/submit', {answers})
    if response['percentage'] == 100:
        print('找到答案！', answers)
        break
```

**修復方式**

加入專用速率限制中介層：

```javascript
// ✅ 修復後：每個 IP 每 15 分鐘最多提交 10 次
const submitLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 10,
  message: { error: '提交次數過多，請稍後再試' }
});

app.post('/api/exams/:id/submit', submitLimiter, ...);
```

### M-1｜輸入值缺乏驗證（中風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A03:2021 – Injection / A04:2021 – Insecure Design |
| **CVSS 評分** | 5.3（中） |
| **受影響端點** | `GET /api/questions?page=&limit=` |
| **修復狀態** | ✅ 已修復 |

**問題描述**

分頁查詢參數 `page` 與 `limit` 未做數值驗證：

```javascript
// ❌ 修復前
const offset = (parseInt(page) - 1) * parseInt(limit);
// parseInt('999999') = 999999 → 一次撈出近百萬筆資料（DoS）
// parseInt('abc')   = NaN    → (NaN - 1) * NaN = NaN → SQL 行為不定
```

**修復方式**

```javascript
// ✅ 修復後：限制上下界、NaN fallback
const pageNum  = Math.max(1, parseInt(page)  || 1);          // 最小 1
const limitNum = Math.min(100, Math.max(1, parseInt(limit) || 20)); // 1 ~ 100
```

### M-2｜缺少安全 HTTP Headers（中風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A05:2021 – Security Misconfiguration |
| **CVSS 評分** | 4.3（中） |
| **受影響範圍** | 所有 HTTP 回應 |
| **修復狀態** | ✅ 已修復 |

**問題描述**

修復前回應標頭缺少以下安全設定：

| Header | 作用 | 修復前 | 修復後 |
|--------|------|--------|--------|
| `X-Content-Type-Options` | 防止 MIME 嗅探攻擊 | ❌ 未設定 | ✅ `nosniff` |
| `X-Frame-Options` | 防止 Clickjacking 攻擊 | ❌ 未設定 | ✅ `SAMEORIGIN` |
| `X-DNS-Prefetch-Control` | 控制 DNS 預取 | ❌ 未設定 | ✅ 已設定 |
| `X-Download-Options` | 防止 IE 直接開啟下載 | ❌ 未設定 | ✅ 已設定 |
| `Referrer-Policy` | 控制 Referrer 資訊 | ❌ 未設定 | ✅ 已設定 |

**修復方式**

```javascript
// ✅ 加入 helmet 套件
const helmet = require('helmet');
app.use(helmet({ contentSecurityPolicy: false }));
```

### M-3｜無 CORS 控制（中風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A05:2021 – Security Misconfiguration |
| **CVSS 評分** | 4.8（中） |
| **受影響範圍** | 所有 API 端點 |
| **修復狀態** | ✅ 已修復 |

**問題描述**

未設定 CORS（跨來源資源共享）政策，任何第三方網站均可在使用者瀏覽器中對本系統 API 發出跨域請求（CSRF 攻擊面）。

**攻擊情境**

```html
<!-- 惡意第三方網站 -->
<script>
  fetch('http://your-server:3000/api/questions', {
    method: 'POST',
    credentials: 'include',
    body: JSON.stringify({ /* 竄改題目 */ })
  });
</script>
```

**修復方式**

```javascript
// ✅ 加入 cors 套件，限定允許來源
const cors = require('cors');
app.use(cors({
  origin: process.env.ALLOWED_ORIGIN || 'http://localhost:3000',
  methods: ['GET', 'POST', 'PUT', 'DELETE']
}));
```

### L-1｜錯誤訊息洩漏（低風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A09:2021 – Security Logging and Monitoring Failures |
| **CVSS 評分** | 3.1（低） |
| **受影響範圍** | 所有路由發生未捕捉例外時 |
| **修復狀態** | ✅ 已修復 |

**問題描述**

未設置全域錯誤處理 middleware，當路由處理器發生未預期例外時，Express 預設行為可能回傳包含 stack trace 的詳細錯誤訊息，洩漏程式碼路徑、套件版本等資訊。

**修復方式**

```javascript
// ✅ 修復後：全域錯誤處理，記錄至 log，不回傳細節
app.use((err, req, res, next) => {
  console.error(`[ERROR] ${req.method} ${req.url}:`, err.message);
  res.status(500).json({ error: '伺服器內部錯誤' });
});
```

### L-2｜多步驟寫入缺乏 Transaction（低風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A04:2021 – Insecure Design（資料完整性） |
| **CVSS 評分** | 2.6（低） |
| **受影響端點** | POST/PUT /api/exams、POST /api/exams/:id/submit |
| **修復狀態** | ✅ 已修復 |

**問題描述**

建立試卷、更新試卷題目、提交作答等操作皆涉及多筆資料庫寫入。修復前未使用 transaction，若伺服器在中途崩潰，可能造成資料不一致（例如試卷建立成功但題目未寫入，或成績記錄不完整）。

**修復方式**

```javascript
// ✅ 修復後：以 db.transaction() 包裝多步驟寫入
const createExam = db.transaction(() => {
  const exam = db.prepare(`INSERT INTO exams ...`).run(...);
  if (question_ids?.length) {
    const ins = db.prepare(`INSERT INTO exam_questions ...`);
    question_ids.forEach((qid, i) => ins.run(...));
  }
  return exam.lastInsertRowid;
});
const id = createExam();
```
